```python
class PrStatus(NamedTuple):
    """Structured PR status data: dots grouped by CI state + unread count."""
    dots_red: list[str]      # FAILURE/ERROR PR dots (may contain OSC8 links)
    dots_pending: list[str]  # PENDING/EXPECTED PR dots
    dots_green: list[str]    # SUCCESS PR dots
    dots_gray: list[str]     # UNKNOWN state PR dots
    unread_count: int        # Unread notification count
```

   - Keep the `_cached_json("pr", ...)` call and node iteration
   - Build the same four dot lists (dots_red, dots_pending, dots_green, dots_gray) with OSC8 links
   - Extract unread_count from cache
   - Return `PrStatus(dots_red, dots_pending, dots_green, dots_gray, unread_count)`
   - Return `None` if no cache, no nodes, or gh unavailable

3. Add `_format_pr_dots(status: PrStatus, include_pr: bool, include_notif: bool) -> str` helper function right after get_pr_status:

   - If `include_notif` and unread_count > 0: append the notification badge `f" {T.notif}...{unread_count}{T.R}"`
   - Return the formatted string (empty string if nothing to show)

4. Update `provider_git` (lines 1360-1412):
   - Instead of `pr_future = pool.submit(get_pr_status)`, keep the same call
   - After getting the result, it's now `PrStatus | None` instead of `str`
   - Replace the ANSI regex block (lines 1404-1411) with:

     ```python
     if pr_data:
     ```

   - Remove the `import re` usage for PR stripping (but keep `re` import if used elsewhere in the file)

   - The old behavior showed these errors inline with the PR dots section

   - If gh errors need to be surfaced, they should go through a dedicated mechanism, not through the PR dot display. For this refactor, returning None is correct and simplifies the interface.
