**DRY-10: Preview segment assembly method**

4. Extract the segment-to-parts loop from `render_preview` (lines 1777-1792) into a private method `_build_preview_line`:

```python
def _build_preview_line(self, segments: list[tuple[str | None, str]], highlight_key: str) -> tuple[str, str]:
    """Build styled preview line and caret indicator from segment list."""
    parts: list[str] = []
    carets: list[str] = []
    for key, text in segments:
        vlen = self._visual_len(text)
        if key is not None:
            parts.append(self._styled(key, text))
            carets.extend(["^" if key == highlight_key else " "] * vlen)
        else:
            parts.append(text)
            carets.extend([" "] * vlen)
    return parts, carets
```

5. Update `render_preview` to use the new method:
```python
parts, caret_chars = self._build_preview_line(segments, cur)
self._append_limits_demo(parts, caret_chars, cur)
preview = "".join(parts)
carets = f"{DIM}{''.join(caret_chars)}{RESET}"
return preview, carets
```

Note: `_append_limits_demo` continues to append directly to the parts/carets lists -- this is fine because it extends the same lists by reference. The preview line builder handles the main segments, and limits demo extends it. This split is correct because limits demo has its own complex per-bar logic that doesn't fit the simple segment pattern.

**DRY-11: Color grid cell rendering**

6. The color grid rendering (render_color_grid) is already a single method with the cell() closure. The "5 sites" from the requirement refers to the 5 grid sections within render_color_grid that call cell() (standard 8, bright 8, cube 6x6x6, grays 24, plus the default row). These are NOT duplicated code -- they are a single cell() function called in different loops for different color ranges.

After examining the code, the actual DRY issue is that render_color_grid is a monolithic method mixing grid layout with cell rendering. Extract the cell rendering closure into a named method for clarity:

```python
def _color_cell(self, n: int, is_bg: bool, sel: int, active: int | None, elem_bg: str) -> str:
    """Render a single color cell with selection and active markers."""
    if is_bg:
        block = f"{bg256(n)}  {RESET}"
    else:
        block = f"{elem_bg}{fg256(n)}[]{RESET}"
    if n == sel:
        return f"{BLINK}{REVERSE}{block}{RESET}"
    if n == active:
        return f"{UNDERLINE}{block}{RESET}"
    return block
```

Then update render_color_grid to use `self._color_cell(n, is_bg, sel, active, elem_bg)` instead of the `cell(n)` closure. Remove the inner `cell` closure. This makes the rendering logic testable and reusable outside the closure scope.
